Methods

onError:errorType:errorMessage: required method

Notified whenever an error has occurred on the record, usually due to an async operation such as a timeout or VersionConflict that can’t be caught sync.

Parameters:
recordName - The name of the record an error occured to
errorType - The error type, such as Event .ACK_TIMEOUT or Event .MESSAGE_DENIED
errorMessage - An error message in english, describing the issue. Do not use this message other than for logging! All checks should be against errorType

onRecordHasProviderChanged:hasProvider: required method

Notified when a listener has confirmed that it will be publishing data. This is extremely useful in the case of records, where you can indicate whether the values shown are cached or up to date.

Parameters:
recordName - The name of the record which has gained/lost a provider
hasProvider - true if provider was found, false otherwise

onRecordDeleted: required method

Notified when the record was deleted, whether by this client or by another.
Once this is called the record object must be cleaned up and a new one created if you wish to continue setting data.

Parameters:
recordName - The name of the deleted record

onRecordDiscarded: required method

Notified once the record was discarded.
Once this is called the record object must be cleaned up and a new one created if you wish to continue setting data.

Parameters:
recordName - The name of the discarded record
