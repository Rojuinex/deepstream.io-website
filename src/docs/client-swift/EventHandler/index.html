Methods

emit:

  • seealso: EventHandler

emit:data:

Emit an event that you want all subscribers, both local and remote to be informed of. You can provide any object that can be serialised to json

Parameters:
eventName - the event name
data - the data to serialise and send with the event

listen:listenListener:

Listen to any subscriptions that have been made on the system that match the provided pattern. If that subscription is found it will give the client the opportunity to accept the role of being the provider, or rejecting it to pass responsibility to someone else.

Parameters:
pattern - The pattern to match, must be valid regex
listenListener - The listener to inform the client when a subscription has been found or removed

subscribe:eventListener:

Subscribes to eventName and notifies the listener via EventListener whenever it occurs locally or remotely

Parameters:
eventName - The event name
eventListener - The eventListener

unlisten:

Remove the listener added via EventHandler, this will remove the provider as the active provider and allow another provider to take its place

Parameters:
pattern - The pattern that has been previously listened to

unsubscribe:eventListener:

Removes the listener added via EventHandler

Parameters:
eventName - The event name
eventListener - The listener that was previous added
